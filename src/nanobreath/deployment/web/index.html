What is this demo doing?

BreathCoach is a real-time browser-based system that detects breath events while you sing and reports phrase-length feedback. The system stacks a small ~15K-parameter neural BreathHead on top of NanoPitch, a frozen pitch-tracking backbone trained for Project 1. The head reads NanoPitch's internal 384-d feature representation and outputs a per-frame probability that the current frame contains an audible breath.

This demo plays a precomputed prediction: a Python script runs NanoPitch + BreathHead on each clip and dumps a JSON of per-frame probabilities. The browser plays the audio and animates the visualization in sync. The live-mic WASM version is the next iteration — the C inference is already proven bit-identical to PyTorch in wasm/test_breath_head.c.

Training (v8): the BreathHead was trained on 75 VocalSet excerpts (32.8 min, 560 breath events) with weak labels generated by the Ruinskiy & Lavner 2007 baseline (the Respiro-en self-training recipe adapted to singing). 150 epochs, plain BCE with pos-weight=3, seq_len=1000, with SpecAugment-style mask + Gaussian noise augmentation on log-mel. Honest full-clip val PR-AUC 0.65, frame F1 0.62, event F1@100ms 0.44, ECE 0.022.

Why BCE and not focal loss? We initially trained with focal loss (v4) and reached PR-AUC 0.66 — but discovered the model was badly miscalibrated (ECE 0.07, max output never crossed 0.4) despite good discrimination. Plain BCE with mild pos-weight gives almost the same discrimination (PR-AUC 0.65) with 3× better calibration (ECE 0.022) and sharply confident outputs (max 0.78). See the project README for the v4-vs-v8-vs-Platt-scaling ablation table.

Demo-set aggregate across the 5 clips above (3.1 minutes of singing total): BreathHead detected 43 breath events, the Ruinskiy DSP baseline detected 23 — the neural model picks up 87% more breaths, mostly the soft inhales that the DSP template-correlation misses. Of course "more events" only matters if they're real; on hand-labeled validation the neural model has recall 0.78 at threshold 0.25 vs Ruinskiy's lower recall on the same data.

Architecture: 2 causal Conv1D layers (kernel 5, dilation 1 and 2) + linear head + sigmoid. The two convolutions give a 130 ms past-context receptive field — enough to detect breath onset before the breath ends. Joint output with NanoPitch's VAD head adds <5% compute over pitch-tracking alone.

The pink markers are the Ruinskiy & Lavner 2007 DSP baseline running on the same audio for comparison. Phrase-length feedback uses whichever event source the precompute script was configured with.
